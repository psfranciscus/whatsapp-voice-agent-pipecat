# Contributing

## Reporting issues

Open an issue with:
- Steps to reproduce
- Expected vs actual behavior
- Pipecat Cloud logs (if applicable)
- Your `pcc-deploy.toml` config (redact secrets)

## Pull requests

1. Fork the repo
2. Create a feature branch
3. Test your changes with a live Pipecat deployment
4. Submit a PR with a clear description of what changed and why

## Code style

- Follow existing Python conventions in `bot.py`
- Use type hints
- Keep functions focused and documented
- Run `uv run ruff check .` before submitting

## Questions

For Kapso-specific questions, check [docs.kapso.ai](https://docs.kapso.ai) or reach out via the Kapso dashboard.

For Pipecat questions, see [Pipecat docs](https://pipecat.daily.co/).
